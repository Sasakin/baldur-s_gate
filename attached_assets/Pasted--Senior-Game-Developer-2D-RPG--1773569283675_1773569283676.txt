# Роль
Ты — Senior Game Developer, специализирующийся на 2D RPG и архитектуре игровых сцен. Твоя задача — спроектировать и написать код для перехода в режим боя.

# Контекст проекта
- Жанр: 2D RPG в стиле Baldur's Gate (изометрия или вид сверху).
- Движок: [УКАЖИ ДВИЖОК, например: Unity / Godot / Custom Engine].
- Язык: [УКАЖИ ЯЗЫК, например: C# / GDScript].
- Текущая проблема: При инициации боя система корректно определяет врагов, но не отображает игрока и его отряд (партию). Участники боя либо не спавнятся, либо находятся за пределами камеры/слоя отрисовки.

# Задача
Реализовать механику перехода в "Боевое Пространство" (Combat Arena), где гарантированно отображаются все участники схватки.

# Технические требования
1. **Смена состояния:**
   - При триггере боя игра должна переключаться в состояние `CombatState`.
   - [ОПЦИОНАЛЬНО: Если у вас отдельная сцена] Загружать сцену боя или активировать боевой UI/слой поверх текущей локации.
   - [ОПЦИОНАЛЬНО: Если одна сцена] Скрывать элементы исследования и активировать боевую камеру.

2. **Спавн участников (Критично):**
   - Написать менеджер боя (`BattleManager`), который принимает список данных об отряде игрока и список врагов.
   - Для каждого члена отряда игрока инстанцировать префаб/спрайт в боевой зоне.
   - Убедиться, что спрайты отряда имеют корректный `Sorting Layer` (не скрыты за фоном или UI).
   - Позиционирование: Расставить отряд игрока и врагов так, чтобы они не перекрывали друг друга при старте (например, по разным сторонам арены).

3. **Камера и Вьюпорт:**
   - Реализовать логику камеры, которая автоматически фокусируется на всех юнитах (`Frame All Units`).
   - Камера должна включать в кадр и отряд игрока, и врагов.
   - Если используется отдельная сцена, настроить `Camera Bounds`.

4. **Передача данных:**
   - Предложить паттерн для передачи текущего состояния отряда (HP, инвентарь, статы) из сцены исследования в сцену боя без потери прогресса (например, через Singleton, ScriptableObject или Event Bus).

# Ожидаемый результат
- Код контроллера боя (`BattleController`).
- Код менеджера спавна юнитов (`UnitSpawner`).
- Логика настройки камеры (`CombatCameraController`).
- Инструкция, где проверить настройки слоев отрисовки (Layers), чтобы отряд стал видимым.

# Ограничения
- Сохранить стилистику Baldur's Gate (партия из нескольких персонажей).
- Минимизировать нагрузку (не пересоздавать объекты, если можно переиспользовать).
- Учесть, что бой может быть пошаговым или Real-Time with Pause (уточни, если важно).
